Visualize Orbitals¶

In [1]:

import os, bz2
import exatomic
from exatomic import gaussian, molcas, nwchem
from exatomic import UniverseWidget as UW

def decompress(fname):
    """Helper function for decompressing example."""
    mod = gaussian
    adir = 'gaussian'
    if fname.startswith('mol'):
        mod = molcas
        adir = 'molcas'
    elif fname.startswith('nw'):
        mod = nwchem
        adir = 'nwchem'
    path = os.path.abspath(os.path.join(os.path.abspath(exatomic.__file__),
                                        "../static/" + adir + "/" + fname + ".bz2"))
    print(path)
    with bz2.open(path) as f:
        if 'fchk' in fname:
            return mod.Fchk(f.read().decode("utf-8"))
        if 'orb' in fname:
            return mod.output.Orb(f.read().decode("utf-8"))
        return mod.Output(f.read().decode("utf-8"))

Provided the outputs are good and the parser is implemented to handle the specifics of the input arguments, this is what the API looks like for orbital viewing

  • First you parse the output (must contain full basis set info and full C matrix)
  • The basis set info is accessible as a basis_set attribute and the C matrix as a momatrix attribute
  • It is an Editor object, which must be converted to a Universe object by .to_universe()
  • The Universe has an add_molecular_orbitals method which by default requires no arguments
  • add_molecular_orbitals optionally can accept the following arguments:
  • field_params (dict): numerical grid parameters (‘rmin’, ‘rmax’, ‘nr’, …) (eg. to make high resolution surfaces)
  • vector (int/list/range): specify the molecular orbitals you would like to view (python is 0-based indexing!)
  • Don’t forget to see the actual docstrings for each function

Gaussian API – should be consistent between QM codes.

The Editor objects deal with file IO and contain parse_methods specific to each dataframe.

In order to add_molecular_orbitals, the Editor must parse the Atom, GaussianBasisSet, BasisSetOrder and MOMatrix dataframes.

The requirements for these dataframes are specified in their docstrings.

The actual interface with the widget occurs on the Universe object, so the Editor must be converted to a Universe.

Therefore, it makes sense to subclass the exatomic.Editor to gain access to the .to_universe() method.
